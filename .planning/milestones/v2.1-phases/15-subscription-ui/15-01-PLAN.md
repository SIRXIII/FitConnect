Add a PRICE_MAP lookup at the top of the handler (after env var reads, before auth), identical to the pattern in stripe-billing-webhook:

```typescript
const PRICE_MAP: Record<string, string> = {
  'pro:month':   Deno.env.get('STRIPE_PRICE_PRO_MONTHLY')!,
  'pro:year':    Deno.env.get('STRIPE_PRICE_PRO_YEARLY')!,
  'elite:month': Deno.env.get('STRIPE_PRICE_ELITE_MONTHLY')!,
  'elite:year':  Deno.env.get('STRIPE_PRICE_ELITE_YEARLY')!,
};
```

Modify the body parsing block (currently lines 26-42) to resolve priceId from EITHER `body.priceId` directly OR `body.tier` + `body.interval` via PRICE_MAP:

```typescript
const body = await req.json();
const priceId = body?.priceId ?? PRICE_MAP[`${body?.tier}:${body?.interval}`];
if (!priceId) {
  return new Response(JSON.stringify({ error: 'Provide priceId or valid tier+interval' }), {
    status: 400,
    headers: { ...corsHeaders, 'Content-Type': 'application/json' },
  });
}
```

This preserves backward compatibility (existing priceId callers still work) while enabling the new tier+interval path.

Do NOT change anything else in the function -- the subscription creation, customer handling, and response are all correct.
