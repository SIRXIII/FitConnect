  <action>
Create the file at the exact path above. Follow the existing stripe-webhook structure precisely:

1. Imports (same order as stripe-webhook):
   - `import 'jsr:@supabase/functions-js/edge-runtime.d.ts';`
   - `import { createClient } from 'https://esm.sh/@supabase/supabase-js@2.49.8';`
   - `import Stripe from 'npm:stripe@14.25.0';` (NOT stripe@17 — version mismatch with other functions)
   - `import { corsHeaders } from '../_shared/cors.ts';`
   - `import { requireEnv } from '../_shared/env.ts';`

2. Outer structure: OPTIONS → POST-only check → try/catch (catch returns 400, not 500 — match existing pattern)

3. Inside try block:
   - `requireEnv` all four secrets: SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, STRIPE_SECRET_KEY, STRIPE_BILLING_WEBHOOK_SECRET
   - Init Stripe: `new Stripe(stripeSecretKey, { apiVersion: '2023-10-16', httpClient: Stripe.createFetchHttpClient() })`
   - Read body: `const body = await req.text()` — MUST be text(), not json(), or HMAC signature breaks
   - Get stripe-signature header; return 400 immediately if absent (before constructEvent)
   - Verify: `stripe.webhooks.constructEvent(body, signature, billingWebhookSecret)` using STRIPE_BILLING_WEBHOOK_SECRET (NOT STRIPE_WEBHOOK_SECRET — different secret, different endpoint)
   - Create adminClient with service_role key (no userClient needed — webhook has no JWT)

4. Build tier lookup map from env (before switch):
   ```typescript
   const TIER_FROM_PRICE: Record<string, 'pro' | 'elite'> = {
     [Deno.env.get('STRIPE_PRICE_PRO_MONTHLY')!]:   'pro',
     [Deno.env.get('STRIPE_PRICE_PRO_YEARLY')!]:    'pro',
     [Deno.env.get('STRIPE_PRICE_ELITE_MONTHLY')!]: 'elite',
     [Deno.env.get('STRIPE_PRICE_ELITE_YEARLY')!]:  'elite',
   };
   ```

5. Create resolveTrainer helper:
   ```typescript
   const resolveTrainer = async (customerId: string) => {
     const { data } = await adminClient
       .from('trainer_profiles')
       .select('id, subscription_status')
       .eq('stripe_customer_id', customerId)
       .maybeSingle();
     return data;
   };
   ```
   If resolveTrainer returns null: log the unknown customerId, return `{ received: true }` with 200 (prevents Stripe infinite retries for non-trainer customers).

6. Idempotency helper (used in all state-writing handlers):
   ```typescript
   const recordEvent = async (trainerId: string, eventType: string, payload: unknown) => {
     const { error } = await adminClient.from('subscription_events').insert({
       trainer_id: trainerId,
       stripe_event_id: event.id,
       event_type: eventType,
       payload,
     });
     return error; // caller checks error.code === '23505'
   };
   ```

7. Switch block — implement all 6 cases:

   **case 'customer.subscription.created'** and **case 'customer.subscription.updated'**:
   - Extract sub as `Stripe.Subscription`
   - Resolve trainer from `sub.customer as string`
   - recordEvent — if 23505 return 200 immediately
   - Derive tier: `TIER_FROM_PRICE[sub.items.data[0].price.id] ?? 'free'`
   - Derive interval: `sub.items.data[0].price.recurring?.interval ?? null` ('month' or 'year')
   - Write to trainer_profiles: subscription_tier, subscription_status (sub.status), subscription_id (sub.id), subscription_interval, trial_ends_at (sub.trial_end ? new Date(sub.trial_end * 1000) : null), current_period_end (new Date(sub.current_period_end * 1000)), cancel_at_period_end (sub.cancel_at_period_end)
   - Filter: `.eq('stripe_customer_id', sub.customer)`

   **case 'customer.subscription.deleted'**:
   - Extract sub as `Stripe.Subscription`
   - Resolve trainer; recordEvent — if 23505 return 200
   - Write: subscription_tier='free', subscription_status='canceled', subscription_id=null, cancel_at_period_end=false, trial_ends_at=null
   - Filter: `.eq('stripe_customer_id', sub.customer)`

   **case 'customer.subscription.trial_will_end'**:
   - Extract sub as `Stripe.Subscription`
   - Resolve trainer (need trainer ID for email lookup)
   - NO DB write — email only
   - Fetch trainer email: join profiles table via trainer.id → `select('email').from('profiles').eq('id', trainer.id).maybeSingle()`
   - Format trial end date from sub.trial_end
   - Send Resend email (non-blocking — wrap in if(resendApiKey) guard, .catch console.error):
     - subject: 'Your FitRush trial ends in 3 days'
     - html body with trial end date and link to add payment: use APP_URL env var for portal link, fallback to 'https://app.fitrush.io/trainer/dashboard?tab=subscription'
   - Always return 200 regardless of email success

   **case 'invoice.paid'**:
   - Extract invoice as `Stripe.Invoice`
   - Resolve trainer from `invoice.customer as string`; recordEvent — if 23505 return 200
   - Write: subscription_status='active', current_period_end=new Date(invoice.lines.data[0]?.period?.end * 1000) (or leave unchanged if no line items)
   - Filter: `.eq('stripe_customer_id', invoice.customer)`

   **case 'invoice.payment_failed'** (BILL-07):
   - Extract invoice as `Stripe.Invoice`
   - Resolve trainer; recordEvent — if 23505 return 200
   - CRITICAL: Only downgrade if subscription_status === 'active' (not 'trialing'):
     ```typescript
     if (trainer.subscription_status === 'active') {
       await adminClient.from('trainer_profiles').update({
         subscription_tier: 'free',
         subscription_status: 'past_due',
       }).eq('stripe_customer_id', invoice.customer);
     }
     // If trialing: do nothing — customer.subscription.deleted fires when trial ends
     ```

   **default**: break (no action for unrecognized events)

8. Return `{ received: true }` with 200 after switch completes.

All adminClient writes use `.eq('stripe_customer_id', customerId)` — never by trainer UUID directly, because webhook events only contain Stripe customer IDs.
