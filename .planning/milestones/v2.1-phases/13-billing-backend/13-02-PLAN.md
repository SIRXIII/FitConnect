  <files>Cenlar demand gt 1-17/supabase/functions/create-subscription/index.ts</files>
  <action>
Create the file using the standard Edge Function skeleton from RESEARCH.md (OPTIONS → POST check → try/catch returning 400).

**Full implementation:**

1. Imports: same 5 imports as stripe-billing-webhook (jsr types, supabase-js@2.49.8, stripe@14.25.0, corsHeaders, requireEnv)

2. Parse request body: `const { priceId } = await req.json()` — after the POST method check, before auth. Return 400 if priceId is missing.

3. Auth: JWT validation using userClient + SUPABASE_ANON_KEY (see interfaces block above). Return 401 if no user.

4. Fetch trainer profile: `adminClient.from('trainer_profiles').select('id, stripe_customer_id, subscription_id').eq('user_id', user.id).maybeSingle()`
   - Return 404 if no trainer profile found

5. Guard: If `trainerProfile.subscription_id` is not null, the trainer already has an active subscription. Return 409: `{ error: 'Trainer already has an active subscription' }` — prevents duplicate subscriptions.

6. Idempotent customer creation:
   - If `trainerProfile.stripe_customer_id` is not null: use it directly
   - If null: call `stripe.customers.create({ email: user.email, metadata: { trainer_id: trainerProfile.id } })`
   - Write new customer ID to DB: `adminClient.from('trainer_profiles').update({ stripe_customer_id: newCustomer.id }).eq('id', trainerProfile.id)`
   - Use the new or existing customer ID going forward

7. Create subscription:
   ```typescript
   const subscription = await stripe.subscriptions.create({
     customer: customerId,
     items: [{ price: priceId }],
     trial_period_days: 30,
     payment_settings: { save_default_payment_method: 'on_subscription' },
     trial_settings: {
       end_behavior: { missing_payment_method: 'cancel' },
     },
   });
   ```
   DO NOT write subscription_tier or subscription_status to the DB here. The customer.subscription.created webhook event fires within milliseconds and is the single writer of subscription state. Writing here causes a race condition.

8. Return 200: `{ subscriptionId: subscription.id, status: subscription.status }`
   - status will be 'trialing' for new trial subscriptions
