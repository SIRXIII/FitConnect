In `create-payment-intent/index.ts`, after the booking record is inserted and before returning the payment intent:
1. Fetch the trainer's `user_id` from the booking
2. Call `send-push-notification` edge function via internal fetch:
   ```typescript
   await fetch(`${Deno.env.get('SUPABASE_URL')}/functions/v1/send-push-notification`, {
     method: 'POST',
     headers: {
       'Authorization': `Bearer ${Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')}`,
       'Content-Type': 'application/json',
     },
     body: JSON.stringify({
       user_ids: [trainerUserId],
       title: 'New Booking',
       body: `${clientName} booked a session with you`,
       data: { type: 'new_booking', booking_id: bookingId },
     }),
   });
   ```
3. Fire-and-forget (do not await or let failure block payment response). Use `.catch(() => {})`.

In `cancel-booking/index.ts`, after booking is cancelled:
1. Determine if cancellation is by trainer or client
2. Send push to the OTHER party:
   - If client cancelled → send to trainer: "A booking was cancelled by [client name]"
   - If trainer cancelled → send to client: "Your session with [trainer name] was cancelled"
3. Also fire-and-forget.

Do NOT push for messages yet (Phase 6 messaging is Supabase Realtime — would need a separate trigger; defer to future enhancement).
