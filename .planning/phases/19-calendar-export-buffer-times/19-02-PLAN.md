Create a new Edge Function at `supabase/functions/calendar-export/index.ts` that handles GET requests with a `token` query parameter.

**Request flow:**
1. Handle OPTIONS (CORS preflight) returning corsHeaders
2. Only accept GET method (return 405 for others)
3. Extract `token` from URL searchParams; return 400 if missing
4. Create admin Supabase client (service role key -- token-based auth, no JWT needed)
5. Look up trainer by `calendar_export_token`:
   ```
   .from('trainer_profiles')
   .select('id, user_id, profiles!trainer_profiles_user_id_fkey(full_name)')
   .eq('calendar_export_token', token)
   .maybeSingle()
   ```
6. Return 404 if no trainer found
7. Query bookings for this trainer:
   ```
   .from('bookings')
   .select('id, notes, status, availability_slots!bookings_slot_id_fkey(start_time, end_time), profiles!bookings_client_id_fkey(full_name)')
   .eq('trainer_id', trainer.id)
   .in('status', ['confirmed', 'pending'])
   .gte('availability_slots.start_time', new Date(Date.now() - 7 * 86400000).toISOString())
   ```
8. Build RFC 5545 .ics string using a `buildICS()` helper function

**buildICS function (inline in same file):**
- Start with VCALENDAR header: VERSION:2.0, PRODID:-//FitRush//Calendar Export//EN, CALSCALE:GREGORIAN, METHOD:PUBLISH, X-WR-CALNAME
- For each booking with a valid availability_slots entry:
  - UID: `{booking.id}@fitrush.app`
  - DTSTAMP: current UTC time in iCal format
  - DTSTART/DTEND: slot start/end in UTC iCal format (`YYYYMMDDTHHMMSSZ`)
  - SUMMARY: `FitRush Session - {clientName}`
  - DESCRIPTION: `Status: {status}` with optional notes
  - STATUS: CONFIRMED or TENTATIVE based on booking status
- Join all lines with `\r\n` (RFC 5545 mandates CRLF, not LF)
- End with END:VCALENDAR

**Response headers:**
- `Content-Type: text/calendar; charset=utf-8`
- `Content-Disposition: attachment; filename="fitrush-schedule.ics"`
- `Cache-Control: public, max-age=300`
- Include corsHeaders spread

**iCal date formatting helper:**
```typescript
function toICalDate(date: Date): string {
  return date.toISOString().replace(/[-:]/g, '').split('.')[0] + 'Z';
}
```

**1. Replace lock_and_mark_slot_on_booking_insert():**

Use `CREATE OR REPLACE FUNCTION` to update the existing trigger function. Keep ALL existing checks (pending status, slot exists, trainer match, not past, not booked), then add buffer enforcement AFTER existing checks, BEFORE the UPDATE:

```sql
-- After existing checks, before UPDATE:
-- Buffer time enforcement
DECLARE v_buffer interval;
DECLARE v_conflict_count int;

SELECT COALESCE(tp.buffer_minutes, 0) * interval '1 minute'
INTO v_buffer
FROM public.trainer_profiles tp
WHERE tp.id = locked_slot.trainer_id;

IF v_buffer > interval '0' THEN
  SELECT COUNT(*) INTO v_conflict_count
  FROM bookings b
  JOIN availability_slots s ON s.id = b.slot_id
  WHERE b.trainer_id = NEW.trainer_id
    AND b.status IN ('pending', 'confirmed')
    AND b.id IS DISTINCT FROM NEW.id
    AND (
      locked_slot.start_time < s.end_time + v_buffer
      AND locked_slot.end_time > s.start_time - v_buffer
    );

  IF v_conflict_count > 0 THEN
    RAISE EXCEPTION 'This slot conflicts with buffer time between sessions. The trainer requires % minutes between bookings.',
      EXTRACT(EPOCH FROM v_buffer) / 60;
  END IF;
END IF;
```

IMPORTANT: The full function must include ALL existing logic from the original trigger (status check, slot existence, trainer match, past check, is_booked check) PLUS the new buffer check. Do not omit any existing validation.
