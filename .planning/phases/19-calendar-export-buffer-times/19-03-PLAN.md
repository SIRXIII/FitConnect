1. Header: "Calendar Export" (serif font, consistent with dashboard card headers)
2. Feed URL section:
   - Display the subscription URL (truncated): `{SUPABASE_URL}/functions/v1/calendar-export?token={token}`
   - "Copy Feed URL" button with Calendar icon from lucide-react
   - On click: `navigator.clipboard.writeText(url)` then `toast.success('Calendar feed URL copied to clipboard')`
   - Helper text: "Paste this URL into Google Calendar or Apple Calendar to subscribe to your schedule."
3. Download section:
   - "Download .ics" button with Download icon
   - On click: fetch the same URL, create Blob, trigger download via temporary anchor element with `download="fitrush-schedule.ics"`
4. Reset token section:
   - "Reset Token" button (smaller, text-ink/40 color, with RefreshCw icon)
   - On click: confirm dialog ("This will invalidate any existing calendar subscriptions. Continue?")
   - If confirmed: call `supabase.rpc('reset_calendar_export_token')`, pass new token to `onTokenReset`
   - Toast: "Calendar token reset. Update your calendar subscription with the new URL."

Use `import.meta.env.VITE_SUPABASE_URL` for the base URL. Import `toast` from `sonner`. Import icons from `lucide-react` (Calendar, Download, RefreshCw, Copy).

Styling: Match existing dashboard card pattern -- `bg-white border border-ink/10 p-6` with rounded corners. Buttons use `border border-ink/20 px-4 py-2 text-[10px] uppercase tracking-[0.2em]` pattern. Use `transition-all duration-300 hover:bg-ink hover:text-paper` for button hover states.

Props: `{ currentBuffer: number; trainerId: string; onBufferChange: (minutes: number) => void }`

Renders:

   - Toast: "Buffer time updated to {X} minutes" (or "Buffer time removed" if 0)
6. Loading state on save button

Styling: Same card pattern as CalendarExportCard. Radio options as horizontal pill-style selectors (like the existing specialty filter pattern). Selected state: `bg-ink text-paper` or `border-accent`.

Both components: `export default ComponentName;` pattern. No barrel files.
