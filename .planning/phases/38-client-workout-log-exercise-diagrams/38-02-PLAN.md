    **WorkoutLogForm.tsx:**
    Props: userId (string), bookingId (string | undefined, optional), onSaved (() => void, callback after successful save).

    State:
    - exercises: array of local form entries, each with: tempId (string for React key), exerciseName (string), exerciseKey (string | null), muscleGroup (string), sets (array of {reps: number | '', weight: number | '', unit: 'lbs' | 'kg'})
    - notes: string (optional session notes)
    - searchQuery: string (for exercise picker)
    - showExercisePicker: boolean
    - saving: boolean
    - weightUnit: 'lbs' | 'kg' (global toggle, default 'lbs')

    Exercise picker section:
    - Text input for search query, filtered via searchExercises(query) from exerciseList.ts
    - Results shown as a scrollable list (max-h-48 overflow-y-auto) with ExerciseDiagram thumbnail (size 32) next to exercise name
    - Tap an exercise to add it to the exercises array with one empty set row
    - "Add Custom Exercise" button at bottom of results, prompts for exercise name, adds with exerciseKey = null
    - After selecting, hide picker and show the exercise in the form

    Per-exercise section (for each exercise in the array):
    - Header row: ExerciseDiagram (size 40), exercise name, remove button (X icon)
    - Set rows: each set has reps input (number, placeholder "Reps"), weight input (number, placeholder "Weight"), unit display showing current weightUnit
    - "Add Set" button below sets (Plus icon, "Add Set" text)
    - Sets default: one empty set row when exercise is added

    Bottom of form:
    - Notes textarea (placeholder "Session notes (optional)", 2 rows)
    - Unit toggle button: shows current unit (lbs/kg), tap to switch
    - Save button: "Save Workout" with loading spinner when saving

    Save logic:
    1. Validate: at least 1 exercise with at least 1 set that has reps > 0
    2. Parse all reps and weight values with parseInt/parseFloat (convert empty strings to 0)
    3. Insert into workout_logs: { client_id: userId, booking_id: bookingId || null, logged_at: new Date().toISOString(), notes: notes.trim() || null }
    4. Get the returned log id
    5. Insert into workout_exercises: for each exercise, { log_id, exercise_name, exercise_key, sort_order (index), sets (JSONB array with unit from weightUnit toggle) }
    6. On success: toast.success('Workout logged!'), call onSaved()
    7. On error: toast.error('Failed to save workout')
    8. Use (supabase as any) pattern for untyped tables

    Use Framer Motion: AnimatePresence for exercise list items entering/exiting. motion.div with layout prop for smooth reordering.

    **WorkoutTab.tsx:**
    Props: userId (string).

    State:
    - logs: WorkoutLogWithExercises[] (fetched from Supabase)
    - loading: boolean
    - showLogForm: boolean (toggle to show/hide WorkoutLogForm)
    - expandedLogId: string | null (which log is expanded to show exercises)
    - page: number (for pagination, starting at 0)
    - hasMore: boolean

    Fetch logic (useCallback + useEffect pattern):
    - Query (supabase as any).from('workout_logs').select('*, workout_exercises(*)').eq('client_id', userId).order('logged_at', { ascending: false }).range(page * 10, (page + 1) * 10 - 1)
    - On page 0, replace logs. On page > 0, append to existing logs.
    - Set hasMore = data.length === 10

    Layout:
    - Header: "Workouts" title, "Log Workout" button (Dumbbell icon from lucide-react)
    - When showLogForm is true, render WorkoutLogForm with onSaved that hides form and refetches page 0
    - Log list: each log card shows:
      - Date (formatted as "Mar 23, 2026"), trainer name if booking_id is linked (fetch from bookings join if available, otherwise show "Standalone workout")
      - Exercise count summary: "5 exercises"
      - Notes preview (truncated to 80 chars) if present
      - Chevron icon indicating expandable
    - Tap a log card: toggle expandedLogId
    - Expanded view (AnimatePresence accordion):
      - For each exercise: ExerciseDiagram (size 36), exercise name, then each set formatted as "10 x 135lbs" using formatSet
      - Notes (full text) if present
    - "Load More" button at bottom when hasMore is true, increments page
    - Empty state when no logs: "No workouts logged yet. Start by tapping 'Log Workout'."
    - Loading state: use existing spinner pattern

    Styling: Tailwind. Cards with rounded-lg border shadow-sm. Consistent with other client dashboard tabs. Mobile-first (full width, no side padding waste).
