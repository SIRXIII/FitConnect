**weekly-payouts/index.ts** — New Edge Function, service-role authenticated (called by pg_cron, not by users):

1. **Auth:** This function is called by pg_cron via net.http_post with the service role key in the Authorization header. Validate that the request has a valid Authorization header. Use the service role key to create adminClient. No user JWT check needed — this is a system function.
2. **Find eligible trainers:** Query all distinct trainer_ids from payments WHERE status = 'succeeded' AND payout_transaction_id IS NULL, joined with bookings. Group by trainer_id and HAVING SUM(trainer_payout) >= 50.
3. **For each eligible trainer:**
   a. Check for existing pending/processing payout_transaction for this trainer. Skip if one exists.
   b. Fetch trainer's stripe_account_id from trainer_profiles. Skip if not connected.
   c. Calculate exact available balance (same query as create-payout).
   d. Insert payout_transactions row: trainer_id, amount, status: 'pending', initiated_by: 'auto'.
   e. Update payments: SET payout_transaction_id = new row ID for the swept payments.
   f. Call stripe.transfers.create({ amount: cents, currency: 'usd', destination: stripe_account_id, metadata: { trainer_id, payout_transaction_id } }).
   g. Update payout_transactions: status = 'processing', stripe_transfer_id = transfer.id.
   h. On Stripe error: Mark payout_transaction as 'failed', reset payout_transaction_id on payments. Log error. Continue to next trainer (do not abort the loop).
   i. Send initiation email via Resend (same pattern as create-payout). Email failure should not block.
4. **Return** 200 with `{ processed: count, failed: failedCount }`.
5. Log each trainer processed for observability.

```typescript
case 'payout.paid': {
  // payout.paid fires on connected accounts when Stripe expects funds in bank.
  // The event.account property identifies which connected account this belongs to.
  const payout = event.data.object as Stripe.Payout;
  const connectedAccountId = (event as any).account as string | undefined;

  if (!connectedAccountId) {
    console.warn('[stripe-webhook] payout.paid missing account field');
    break;
  }

  // Find the trainer by stripe_account_id
  const { data: trainer } = await adminClient
    .from('trainer_profiles')
    .select('id, user_id')
    .eq('stripe_account_id', connectedAccountId)
    .maybeSingle();

  if (!trainer) {
    console.warn('[stripe-webhook] payout.paid: no trainer for account', connectedAccountId);
    break;
  }

  // Find the most recent 'processing' payout_transaction for this trainer
  const { data: payoutTx } = await adminClient
    .from('payout_transactions')
    .select('id, amount')
    .eq('trainer_id', trainer.id)
    .eq('status', 'processing')
    .order('created_at', { ascending: false })
    .limit(1)
    .maybeSingle();

  if (payoutTx) {
    // Mark payout as completed
    await adminClient
      .from('payout_transactions')
      .update({ status: 'completed', updated_at: new Date().toISOString() })
      .eq('id', payoutTx.id);

    // Get trainer's email for completion notification
    const { data: profile } = await adminClient
      .from('profiles')
      .select('email')
      .eq('id', trainer.user_id)
      .maybeSingle();

    if (profile?.email) {
      // Send completion email via Resend directly (service role context, no need for separate Edge Function call)
      const resendApiKey = Deno.env.get('RESEND_API_KEY');
      if (resendApiKey) {
        const amount = Number(payoutTx.amount).toFixed(2);
        await fetch('https://api.resend.com/emails', {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${resendApiKey}`,
            'Content-Type': 'application/json',
          },
          body: JSON.stringify({
            from: 'FitRush <noreply@resend.dev>',
            to: [profile.email],
            subject: 'Your FitRush payout has arrived',
            html: `<p>Your payout of $${amount} has arrived in your bank account.</p>`,
          }),
        }).catch(err => console.error('[stripe-webhook] Resend error:', err));
      }
    }
  }
  break;
}
```

Place this case BEFORE the `default: break;` in the switch statement.
