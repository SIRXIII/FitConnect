### Human Verification Required

#### 1. Avatar Upload Flow

**Test:** Navigate to `/client/passport` and click the avatar circle to upload a photo.
**Expected:** Photo compresses via `compressImage`, uploads to the `avatars/` Supabase Storage bucket, the preview updates immediately, a "Photo uploaded!" toast appears, and the progress ring advances (avatar field counts as complete).
**Why human:** Requires a real browser environment, file input interaction, and Supabase Storage credentials.

#### 2. Auto-Save Toast Deduplication

**Test:** On the ClientPassport page, rapidly blur multiple text fields (bio then age) in quick succession.
**Expected:** At most one "Saved" toast is visible at any time — the `{ id: 'profile-save' }` option on `toast.success` should replace rather than stack multiple toasts.
**Why human:** Toast timing and visual deduplication behavior requires live UI interaction.

#### 3. GoalRankPicker Capacity Cap

**Test:** Add 3 goals to the ranked list via GoalRankPicker, then attempt to click a 4th goal in the available pool.
**Expected:** Remaining pool pills display with `opacity-40 cursor-not-allowed` styling and clicking does nothing (the `isFull` guard prevents adding).
**Why human:** Visual styling state and click-disabled behavior require browser rendering.

#### 4. ClientSummaryCard in Go Live Queue (Trainer View)

**Test:** As a trainer with a pending booking request from a client who has set `fitness_level`, `health_conditions`, and `intensity_preference`, open the Go Live request queue.
**Expected:** The `BookingRequestCard` shows the collapsed `ClientSummaryCard` row with the fitness level badge, primary goal badge, a colored dot + intensity label, and AlertTriangle icons for each condition (max 2 visible + "+N" overflow).
**Why human:** Requires a real database row with Phase 23.1 fields populated in `client_profiles` and an associated pending `booking_request`.

#### 5. ClientSummaryCard Expand on TrainerBookings

**Test:** On the TrainerBookings page, find a booking with a client who has Phase 23.1 profile data and click the "Details" text link on the ClientSummaryCard.
**Expected:** A smooth height-animation expansion reveals numbered ranked goals, age/weight (if set), health notes (if any), and workout types as comma-separated labels. Clicking "Hide" collapses it.
**Why human:** Framer Motion AnimatePresence height animation requires browser rendering; test data requires Phase 23.1 fields to be populated.
