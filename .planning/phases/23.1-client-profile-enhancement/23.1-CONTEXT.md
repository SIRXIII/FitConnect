# Phase 23.1: Client Profile Enhancement - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Clients can view and edit their full profile (photos, personal stats, fitness level, health conditions, workout intensity preferences) after initial onboarding. Trainers can see a summary card of client data before sessions. The existing ClientPassport page is extended to become the single profile editing hub.

</domain>

<decisions>
## Implementation Decisions

### Profile editing experience
- Extend the existing ClientPassport page (not a new page)
- Scrollable sections layout: Personal Info, Health, Preferences (one long page, labeled sections)
- Auto-save on blur — each field saves when user taps out, subtle toast confirms
- Progress ring at top showing completion percentage (e.g. "75% complete") with hints on missing fields
- Fields from ClientOnboarding (age, weight, height, body type, fitness level, goals, workout types, health notes) become editable here

### Health & intensity controls
- Structured checklist + free text for health conditions
  - Common conditions as checkboxes: Back pain, Knee injury, Asthma, Heart condition, Pregnancy, Diabetes
  - Free-text "Other notes" field for anything not covered
- Workout intensity preference: visual slider with 3 stops
  - Light to Moderate to Intense
  - Each stop shows brief description (e.g. "Light: focus on form and mobility")
  - Color shifts from green to amber to red
- Fitness goals: drag-to-rank top 3 priorities (not just multi-select)

### Trainer visibility of client data
- Summary card (condensed view): photo, fitness level, primary goal, health flags as icons/badges, intensity preference
- Details available on tap/expand
- Card appears on booking detail page AND in Go Live request queue (before accepting)
- All fields visible to booked trainers by default — no per-field privacy toggles

### Photo gallery
- Avatar only — keep the existing single avatar upload from ClientPassport
- No progress photos or gallery for this phase

### Claude's Discretion
- Exact section spacing and visual styling within FitRush brand
- Health condition icon/badge design
- Drag-to-rank implementation approach (native drag or button-based reorder)
- Progress ring animation and color scheme
- Auto-save debounce timing

</decisions>
