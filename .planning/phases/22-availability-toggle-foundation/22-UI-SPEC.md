## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FDFCFB (`paper`) | Page background, sticky header bar background |
| Secondary (30%) | #1A1A1A (`ink`) at various opacities | Cards, borders, text, offline state |
| Accent (10%) | #C5A059 (`accent`) | Reserved-for list below |
| Destructive | `ink` at full opacity + red-600 (#DC2626) | Auto-decline warning, irreversible actions |

Accent (`#C5A059`) reserved for:
- Live toggle active state — green accent bar at top of sticky header when trainer is LIVE
- "LIVE" status indicator dot on trainer cards in search results ("Live Now" badge)
- Sleep timer countdown text when timer is set and running
- Countdown-to-auto-decline text in the request queue card
- "Instant Book" badge text on trainer profile cards
- Save 20% / promotional highlights (existing pattern — no new usage added)

**Live state color exception:** The sticky header bar uses a 2px top border in `#22C55E` (Tailwind green-500) when trainer is online — this is a semantic live-status color, not a new accent. The toggle pill itself uses `bg-green-500 text-white` for the ON state and `bg-ink/10 text-ink/40` for OFF state. The warm-up state uses `bg-amber-400` (Tailwind amber-400).

Destructive: `#DC2626` (red-600) is used only for the auto-decline warning label in the request queue card and the "Going offline with active bookings" warning copy. All other destructive actions use ink + confirmation copy.

Source: index.css @theme, TrainerCard.tsx (accent for price/rating), BillingToggle.tsx (ink selected state)

## Interaction Contracts

### Sticky Availability Header Bar

- Position: fixed below global nav, always visible on trainer dashboard (`top-16` or `top-0` with `z-40`)
- Height: 64px
- Background: `bg-paper border-b border-ink/10`
- Live state top border: `border-t-2 border-green-500` (2px green bar at very top of header)
- Contents (left to right): toggle pill | status label + countdown | sleep timer pills
- Toggle pill: 44px min-height, `rounded-full` 28px wide, slides between OFF (left) / ON (right)

### Toggle States

| State | Background | Label | Border |
|-------|-----------|-------|--------|
| OFFLINE | `bg-ink/10` | "OFFLINE" ink/40 | none |
| GOING LIVE (0–5s) | `bg-amber-400` | "GOING LIVE..." ink | amber pulse ring |
| LIVE | `bg-green-500` | "YOU ARE LIVE" ink | green-500 top border on header |

- Warm-up animation: 5-second progress bar fills amber header stripe left-to-right using framer-motion `animate={{ width: '100%' }}` with `duration: 5` linear easing
- After warm-up completes: cross-fade to green using `AnimatePresence` (200ms opacity transition)
- Toggle tap during warm-up: cancels warm-up, returns to OFFLINE immediately

### Sleep Timer Pills

- Render only when toggle is LIVE or in warm-up
- Pills: "1hr" | "2hr" | "4hr" | "EOD"
- Selected pill: `bg-ink text-white` (matches BillingToggle selected state)
- Unselected: `text-ink/40 hover:text-ink`
- Tapping active countdown extends timer: pills re-appear, new selection adds to remaining time
- EOD resolves to midnight in `Intl.DateTimeFormat` local timezone

### Request Queue Card

- Container: `bg-paper border border-ink/10` with `space-y-4` padding `p-6`
- Client avatar: 40px circle, `object-cover`
- Accept button: `bg-ink text-white` full-width bottom, 44px height
- Decline button: `border border-ink/10 text-ink/40 hover:text-ink` full-width, 44px height
- Countdown-to-auto-decline: `text-accent` when >10 minutes, `text-red-600` when <=5 minutes
- Max 5 cards in queue; when full, new client sees inline message (not toast): "This trainer is reviewing requests"

### Double-Booking Race Error

- Delivery: sonner toast, `toast.error()` variant
- Duration: 4000ms (default)
- Copy: "This slot was just booked. Pick another time." (see Copywriting Contract)

### First-Time Use Tooltip

- Trigger: first render if `availability_status` has never been set (null in DB)
- Placement: below the toggle pill, arrow pointing up
- Style: `bg-ink text-paper text-[10px] uppercase tracking-[0.2em]` with `p-3`
- Dismiss: tap anywhere outside or tap "Got it" link within tooltip
- Does not block toggle interaction

### Availability Indicator on Trainer Cards (Client View)

- Green dot: `w-2 h-2 rounded-full bg-green-500 animate-pulse` inline before "Live Now" label
- "LIVE NOW" badge: `text-[9px] uppercase tracking-[0.2em] text-green-600 font-medium`
- "Instant Book" badge: `text-[9px] uppercase tracking-[0.2em] text-accent font-medium` below trainer name
- "Request to Book" badge: `text-[9px] uppercase tracking-[0.2em] text-ink/40 font-medium`
- Live trainers sort above non-live in search results (visual priority is sort-order, not style-only)

### 10-Minute Warning Notification

- Delivery: sonner toast `toast.warning()` with action button
- Copy: "Availability expires in 10 min. Extend?" with "Extend" action that reopens sleep timer pills
- Duration: persistent (requires user action or manual dismiss)

## Accessibility Notes

- Live toggle must have `aria-pressed` (true/false) and `aria-label="Go live"` / `aria-label="Go offline"`
- Countdown must use `aria-live="polite"` so screen readers announce timer updates without interrupting
- Destructive "Go Offline Anyway" button must be visually distinct (`border border-red-600 text-red-600`) to prevent accidental tap
- All touch targets minimum 44px height (iOS HIG requirement — Capacitor iOS deployment)
- Sleep timer pills keyboard-accessible via `role="group"` with `aria-label="Sleep timer duration"`
