**1. Create `useAvailabilitySession.ts`** at `Cenlar demand gt 1-17/src/hooks/useAvailabilitySession.ts`:

State machine with 3 local UI states: `'offline' | 'going_live' | 'live'`

The hook should:
- Accept `trainerProfile` from auth store (or null)
- Maintain local state `uiStatus: 'offline' | 'going_live' | 'live'` synced from `trainerProfile.availability_status`
- `goLive(bookingMode: 'instant' | 'request', timerDuration: number | 'eod' | null)`: Sets `uiStatus = 'going_live'`, waits 5 seconds (via setTimeout). After 5s, writes to DB: `availability_status: 'live', booking_mode, sleep_timer_expires_at, availability_session_started_at: new Date().toISOString()`. Then sets `uiStatus = 'live'`. Returns a cancel function.
- `goOffline()`: Writes `availability_status: 'offline', sleep_timer_expires_at: null, availability_session_started_at: null` to DB. Sets `uiStatus = 'offline'`.
- `setSleepTimer(duration: number | 'eod')`: Computes `sleep_timer_expires_at` as ISO string. For numeric hours: `new Date(Date.now() + duration * 3600 * 1000)`. For `'eod'`: `const eod = new Date(); eod.setHours(23, 59, 59, 0); return eod;`. Writes to DB via `supabase.from('trainer_profiles').update({ sleep_timer_expires_at }).eq('id', trainerProfile.id)`.
- `extendTimer(duration: number | 'eod')`: If timer currently set, compute new expiry as `max(current_expires_at, now()) + duration` (adds to remaining, not from now). Write to DB.
- `cancelWarmup()`: If `uiStatus === 'going_live'`, clear the 5s timeout, set `uiStatus = 'offline'`, do NOT write to DB (since DB was never updated during warmup — per Pitfall 5 in RESEARCH.md).
- Countdown state: `secondsRemaining` computed via `setInterval(tick, 1000)` from `trainerProfile.sleep_timer_expires_at`. Display format computed as `const hours = Math.floor(secondsRemaining / 3600); const minutes = Math.floor((secondsRemaining % 3600) / 60);`.
- 10-minute warning: When `secondsRemaining === 600`, fire `toast.warning("Availability expires in 10 min. Extend Timer?", { duration: Infinity, action: { label: 'Extend Timer', onClick: () => setShowExtendPills(true) } })` using sonner.
- Realtime subscription: Subscribe to `postgres_changes` on `trainer_profiles` filtered by `id=eq.${trainerProfile.id}`, update local state from payload.new on external changes (e.g., pg_cron setting status to offline). Use channel name `trainer-availability-${trainerProfile.id}`. Clean up on unmount.
- After DB write succeeds, call `useAuthStore.getState().fetchProfile(trainerProfile.user_id)` to refresh trainerProfile in Zustand store.

Return type:
```typescript
{
  uiStatus: 'offline' | 'going_live' | 'live';
  bookingMode: 'instant' | 'request';
  secondsRemaining: number;
  countdownDisplay: string; // "2h 14m remaining" or ""
  showExtendPills: boolean;
  setShowExtendPills: (v: boolean) => void;
  goLive: (bookingMode: 'instant' | 'request', timer: number | 'eod' | null) => void;
  goOffline: () => Promise<void>;
  cancelWarmup: () => void;
  setSleepTimer: (duration: number | 'eod') => void;
  extendTimer: (duration: number | 'eod') => void;
}
```
