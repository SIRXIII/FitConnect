**Flow:**
1. Create adminClient with service role key
2. Determine which trainers to sync:
   - If called by service role (cron): fetch ALL active connections: `adminClient.from('google_calendar_connections').select('*').eq('is_active', true)`
   - If called by authenticated user: fetch only their connection
3. For each connection:
   a. Get valid access token via `getValidAccessToken(connection, adminClient)`. On 'gcal_token_revoked', skip this trainer (already marked inactive by helper), continue to next.
   b. Compute time window: timeMin = now ISO, timeMax = now + 28 days ISO
   c. Call `listGcalEvents(accessToken, timeMin, timeMax)` — this already excludes fitrush-tagged events (Pitfall 4 handled in helper)
   d. Filter events: only include events with valid start.dateTime and end.dateTime (skip all-day events which have start.date instead of start.dateTime)
   e. Delete existing gcal_blocked_slots for this trainer that are older than timeMin (cleanup past events): `adminClient.from('gcal_blocked_slots').delete().eq('trainer_id', connection.trainer_id).lt('ends_at', timeMin)`
   f. Upsert current external events into gcal_blocked_slots: for each event, upsert { trainer_id: connection.trainer_id, gcal_event_id: event.id, gcal_summary: event.summary || null, starts_at: event.start.dateTime, ends_at: event.end.dateTime, synced_at: new Date().toISOString() } with onConflict: 'trainer_id,gcal_event_id'

   h. Update last_sync_at on connection: `adminClient.from('google_calendar_connections').update({ last_sync_at: new Date().toISOString() }).eq('trainer_id', connection.trainer_id)`
4. Return 200 with { success: true, synced_trainers: count }

Standard CORS handling. POST only. Wrap entire loop in try/catch so one trainer's failure doesn't stop others.
