    Create migration file `20260320100000_location_notifications.sql`:

    1. CREATE TABLE `client_notification_preferences`:
       - `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
       - `user_id uuid NOT NULL UNIQUE REFERENCES profiles(id) ON DELETE CASCADE`
       - `notif_enabled boolean NOT NULL DEFAULT false`
       - `area_label text` (human-readable from Places Autocomplete)
       - `area_lat double precision`
       - `area_lng double precision`
       - `notif_radius_miles int NOT NULL DEFAULT 5 CHECK (>= 1 AND <= 10)`
       - `created_at timestamptz NOT NULL DEFAULT now()`
       - `updated_at timestamptz NOT NULL DEFAULT now()`

    2. Enable RLS. Create policy "client manages own notif prefs" FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id).

    3. Create updated_at trigger using the existing `moddatetime` pattern or a simple trigger function (check what the codebase uses for updated_at).

    4. Create `notify_nearby_clients()` SECURITY DEFINER function SET search_path = public:
       - Guard: only fire when NEW.availability_status = 'live' AND OLD.availability_status != 'live'
       - Guard: only fire when NEW.active_location_id IS NOT NULL
       - Fetch trainer name from profiles join, rate from trainer_profiles
       - Fetch active workout location address + lat/lng from workout_locations
       - Loop over client_notification_preferences WHERE notif_enabled = true AND area_lat/lng IS NOT NULL
       - PostGIS check: `extensions.ST_DWithin(extensions.ST_MakePoint(trainer_lng, trainer_lat)::extensions.geography, extensions.ST_MakePoint(pref.area_lng, pref.area_lat)::extensions.geography, pref.notif_radius_miles * 1609.34)`
       - Daily cap: COUNT(*) from notifications WHERE user_id = pref.user_id AND type = 'trainer_live_nearby' AND created_at > now() - INTERVAL '24 hours'; skip if >= 3
       - Trainer cooldown: COUNT(*) from notifications WHERE user_id = pref.user_id AND type = 'trainer_live_nearby' AND link LIKE '%' || NEW.id || '%' AND created_at > now() - INTERVAL '4 hours'; skip if > 0
       - INSERT into notifications: type = 'trainer_live_nearby', title = '{name} just went live nearby', message = '{address} -- ${rate}/hr', link = '/search?trainer=' || NEW.id
       - Use fully qualified `extensions.ST_DWithin`, `extensions.ST_MakePoint` (Pitfall 3 from research)

    5. CREATE TRIGGER `trainer_went_live_notify_clients` AFTER UPDATE ON trainer_profiles FOR EACH ROW EXECUTE FUNCTION notify_nearby_clients();

    6. Create `src/types/notifications.ts` with:

       ```typescript
         notif_radius_miles: number;
         created_at: string;
         updated_at: string;
       }
       ```
