Create `Cenlar demand gt 1-17/supabase/functions/waitlist-signup/index.ts` as a NEW public Edge Function (no auth check).

The function must:

1. Handle OPTIONS preflight with `corsHeaders` from `../_shared/cors.ts`
2. Reject non-POST with 405
3. Parse JSON body, extract `email` field
4. Validate email server-side: must be string, non-empty, match basic email regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
5. Return 400 with `{ error: 'Valid email is required' }` if invalid
6. Create Supabase client using `requireEnv('SUPABASE_URL')` and `requireEnv('SUPABASE_SERVICE_ROLE_KEY')` (service-role bypasses RLS)
7. Insert `{ email: email.trim().toLowerCase() }` into `email_subscribers`
8. On PostgreSQL error code `23505` (unique constraint violation): return 200 with `{ success: true }` -- do NOT reveal that the email already exists
9. On other insert errors: throw to hit the catch block
10. After successful insert, send confirmation email via Resend API:
    - Use `Deno.env.get('RESEND_API_KEY')` -- if not set, skip email (dev mode)
    - POST to `https://api.resend.com/emails`
    - from: `FitRush <noreply@resend.dev>`
    - to: the submitted email
    - subject: `You are on the FitRush early access list.`
    - html: branded HTML with FitRush name, gold accent `#C5A059`, message: "Welcome to FitRush. You are on the early access list. We will reach out when it is time." plus "Elite Fitness Marketplace" footer
    - Email failure is NON-FATAL: log with `console.error` but do NOT throw
11. Return 200 with `{ success: true }`
12. Catch block returns 500 with error message
