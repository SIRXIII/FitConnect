| Token | Value | Usage |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 400 (regular, serif Cormorant Garamond) | 1.2 |
| Display | 28px | 300 (light, serif italic Cormorant Garamond) | 1.2 |

Notes:
- Section headings ("Session Notes", "Workout Log", tab labels) use Label at 12px with `uppercase tracking-[0.2em]` — this is the established app pattern (`text-xs uppercase tracking-[0.2em] text-ink/40`).
- Progress timeline dates use Label at 12px, weight 400, `text-ink/40`.
- Exercise names in the workout log use Body at 14px, weight 400.
- Numeric inputs (sets, reps) use Body at 14px, weight 500, center-aligned.
- Chart axis labels use 11px / weight 400 — Recharts default, do not override.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FDFCFB` (`var(--color-paper)`) | Page background, textarea background, section surfaces |
| Secondary (30%) | `rgba(26,26,26,0.05)` (`bg-ink/5`) | Completed booking cards, expandable notes section background, timeline entry rows |
| Accent (10%) | `#C5A059` (`var(--color-accent)`) | Section icon accent, Save button border/text, chart line for sessions-per-week |
| Destructive | `rgb(239,68,68)` (`text-red-600`) | Lock icon and "Editing window closed" state text; no destructive actions in this phase |

Accent reserved for:
- The Save Notes button (border + text, not filled)
- The sessions-per-week line in the Recharts line chart
- Section header icons that indicate a logged/completed state (e.g. checkmark after auto-save)

The total-sets-per-week chart line uses `rgba(26,26,26,0.4)` (`text-ink/40` equivalent) — a muted secondary line, not accent.

Source: `src/index.css` token definitions + `AnalyticsTab.tsx` chart color patterns (`chartColors.stroke = '#2d2d2d'`).

## Component Inventory

### SessionNotesSection (new — added to completed booking cards in TrainerBookings)

- Triggered by: clicking "Add Session Notes" expand button on a completed booking card
- Expand/collapse animation: `AnimatePresence` + `motion.div` with `initial={{ height: 0, opacity: 0 }}` / `animate={{ height: 'auto', opacity: 1 }}` — same pattern as `ClientSummaryCard.tsx`
- Textarea: full-width, `border border-ink/10 bg-paper p-4 text-sm resize-none min-h-[96px]` — no outline ring, uses app's minimal border style
- Auto-save trigger: `onBlur` — same as `ClientPassport.tsx` `saveField()` pattern
- Save feedback: Sonner toast, duration 2000ms, message "Notes saved." — no icon needed
- Locked state (>24hrs post-session): textarea replaced with `<p>` in `text-ink/60 text-sm`, lock icon (lucide `Lock` at 12px, `text-ink/30`) and caption "Editing closed 24 hours after the session" in 11px `text-ink/30`
- Expand button label (pre-notes): "Add session notes"
- Expand button label (post-notes): "Edit session notes" (within 24hr window) or "View session notes" (locked)

### WorkoutLogSection (new — below SessionNotesSection on completed booking cards)

- Heading: `text-xs uppercase tracking-[0.2em] text-ink/40` — "Workout Log"
- Exercise row layout: `flex gap-2 items-center` — exercise name input (flex-1) + sets numeric input (w-16) + "×" separator text (text-ink/30) + reps numeric input (w-16) + remove button (lucide `X` at 12px, `text-ink/30 hover:text-red-500`)
- Exercise name input: `border-b border-ink/10 bg-transparent px-0 py-1 text-sm focus:outline-none focus:border-ink/30` — underline-only style, no box
- Sets/reps inputs: `border border-ink/10 text-center w-16 h-9 text-sm bg-paper` — boxed, compact
- "Add exercise" button: `flex items-center gap-1 text-xs text-ink/40 hover:text-ink/70 mt-2` with lucide `Plus` at 12px — subtle text button, no border
- Save trigger: explicit "Save workout" button (`border border-accent text-accent text-xs uppercase tracking-[0.2em] px-4 py-2 hover:bg-accent/5`) — workout data saved on button press, not on blur
- Save feedback: Sonner toast, duration 2000ms, message "Workout saved."
- Empty state (no exercises yet): single line "No exercises logged yet." in `text-xs text-ink/30 italic`

### ClientProgressTab (new — tab added to ClientDashboard)

- Tab label: "Progress" — follows existing tab pattern with `text-xs uppercase tracking-[0.2em]`
- Layout: two stacked sections separated by `xl` (32px) gap:
  1. Session timeline list
  2. Progress chart (Recharts LineChart)

### SessionTimelineList (new — inside ClientProgressTab)

- Each entry: `flex gap-4 items-start border-b border-ink/5 py-4`
- Left column (date): `text-xs text-ink/40 w-20 shrink-0 pt-0.5` — formatted as "Mar 14" (short month + day)
- Right column: trainer name in `text-sm font-medium text-ink` + notes summary in `text-sm text-ink/60 mt-1 line-clamp-2` + exercise count in `text-xs text-ink/30 mt-1` (e.g., "4 exercises")
- Notes summary: 2 lines max via `line-clamp-2` — no expand in timeline (full notes visible on MyBookings)
- No avatar in timeline rows — keeps the list scannable
- Loading state: reuse `BookingCardSkeleton` pattern — 3 skeleton rows

### ProgressLineChart (new — inside ClientProgressTab)

- Recharts `LineChart` inside `ResponsiveContainer` with `height={200}`
- Two lines: sessions/week (accent `#C5A059`, strokeWidth 1.5) + sets/week (`rgba(45,45,45,0.4)`, strokeWidth 1.5)
- No `CartesianGrid` — clean minimal look per CONTEXT.md "no grid clutter"
- `XAxis`: week labels ("Mar 10", "Mar 17", etc.), `tick={{ fontSize: 11, fill: 'rgba(26,26,26,0.4)' }}`
- `YAxis`: hidden (`hide={true}`) — values surfaced only in tooltip
- `Tooltip`: custom minimal style — `bg-paper border border-ink/10 text-xs p-2`
- Legend: simple two-item text legend below chart, `text-xs text-ink/40`, colored dot + label
- Time range: 12 weeks (3 months) — default and only range for this phase (Claude's discretion: 12 weeks gives enough history to show trends without sparse early data for new users)

### SessionNotesClientView (new — added to past booking cards in MyBookings)

- Read-only display (client cannot edit trainer's notes)
- Expand/collapse same `AnimatePresence` pattern as above
- Notes text: `text-sm text-ink leading-relaxed`
- Exercises list: `space-y-1 mt-2` — each row `text-sm text-ink/70` formatted as "Squats — 3 × 12"
- Expand button label: "View session notes" (only shown when notes exist)
- No expand button if trainer left no notes — section is hidden entirely

## Animation Contract

| Element | Animation |
|---------|-----------|
| Notes section expand | `height: 0 → auto`, `opacity: 0 → 1`, duration 200ms ease-out (AnimatePresence + motion.div) |
| Notes section collapse | `height: auto → 0`, `opacity: 1 → 0`, duration 150ms ease-in |
| Exercise row add | `opacity: 0, y: 4 → opacity: 1, y: 0`, duration 150ms |
| Exercise row remove | `opacity: 1 → 0`, duration 100ms, row collapses out |
| Progress tab mount | inherited from existing tab switching pattern — no special animation |
| Toast | Sonner default slide-in — do not override |
