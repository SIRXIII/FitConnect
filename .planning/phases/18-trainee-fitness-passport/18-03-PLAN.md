
**2. Update the Supabase query** (around line 51):
Add `client_profiles` to the nested select. The bookings table has `client_id` which references `profiles.id`, and `client_profiles` has `user_id` which also references `profiles.id`. We need to join through the client_id.

Update the select string to include:
```
client_profiles!client_profiles_user_id_fkey (
  bio,
  fitness_goals,
  workout_types,
  training_frequency,
  health_notes,
  fitness_level
)
```

BUT: Supabase's PostgREST cannot directly join client_profiles through bookings because there's no direct FK from bookings to client_profiles. The FK chain is: bookings.client_id -> profiles.id <- client_profiles.user_id.

**Workaround**: Fetch client_profiles in a second query after getting bookings. After the main bookings query resolves:
- Collect unique client IDs from `bookings.map(b => b.profiles?.id).filter(Boolean)`
- Fetch from `client_profiles` where `user_id` is in that list: `supabase.from('client_profiles').select('user_id, bio, fitness_goals, workout_types, training_frequency, health_notes, fitness_level').in('user_id', clientIds)`
- Create a Map<string, ClientProfile> keyed by user_id
- Merge into bookings array by matching `booking.profiles.id` to `clientProfile.user_id`
