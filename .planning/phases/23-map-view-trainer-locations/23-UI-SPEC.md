## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 (regular) | 1.5 | Inter (sans) |
| Label | 10px | 600 (semibold), uppercase, tracking-[0.2em] | 1 | Inter (sans) |
| Heading | 24px | 400 (regular) | 1.2 | Cormorant Garamond (serif) |
| Display | 20px | 400 italic | 1.2 | Cormorant Garamond (serif, prices/rates) |

Pattern from TrainerCard: `text-2xl serif` for name, `text-[10px] uppercase tracking-widest` for labels. The serif typeface provides sufficient visual distinction from Inter without font-light. All labels use uppercase + wide tracking — apply consistently on info card, chip toggles, and location manager.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FDFCFB` (`--color-paper`) | Map container background, info card surface, bottom sheet background |
| Secondary (30%) | `#1A1A1A` (`--color-ink`) at `ink/5`–`ink/10` opacity | Cards, filter panels, chip toggle borders, map overlay controls |
| Accent (10%) | `#C5A059` (`--color-accent`) | Rate display, star ratings, discount badge, Award icon, Elite pin ring, CTA "Book" button background |
| Destructive | `#EF4444` (Tailwind `red-500`) | Delete location action only |

Accent reserved for: rate/price text, rating stars, discount percentage badge, Elite trainer pin gold ring/border, "Book" button on info card.

Location type color-coding (pin and chip):
- Gym: `blue-600` (#2563EB)
- Park: `green-600` (#16A34A)
- In-Home: `amber-600` (#D97706)

Live status: `green-500` (#22C55E) pulsing dot — matches existing `LiveNowBadge` pattern.

## Interaction Contract

### Map View Toggle

- Toggle button at top of SearchSection, replacing any existing view-mode selector
- State: `viewMode: 'list' | 'map'` — persists for session duration, defaults to `'list'`
- All active filters (specialty, price, distance) remain applied when switching views
- Transition: AnimatePresence fade (200ms) between list and map containers

### Pin Behavior

- Pins render only for trainers with `availability_status === 'live'` (MAP-04)
- Clustered pin: dark ink circle with white count number; clicking zooms in to reveal individual pins; zoom threshold determined by `@vis.gl/react-google-maps` default clustering
- Individual pin: 40×40px tap area minimum; visual pin body 28×36px
- Clicking pin: closes any open info card, opens new info card anchored near the pin; clicking same pin again closes card
- Clicking outside any pin or info card X button: closes info card
- Info card Book button: navigates to `/book/:trainerId` (existing booking wizard)

### "Search This Area" Button

- Appears after user pans map 100+ pixels from the initial center
- Disappears after user clicks it (re-query fires)
- Position: horizontally centered, 16px from top of map container
- While re-querying: button text changes to "Searching..." and is disabled

### Distance Filter

- Distance slider (1–25 miles) in existing filter panel
- When slider value changes: draws RadiusCircle on map centered on client's location dot
- Pins outside the radius are hidden; RadiusCircle uses `blue-600` at 15% opacity fill, 1px `blue-600` border

### Geolocation

- Permission request fires when client switches to map view for the first time
- If granted: map centers on client location, ClientLocationDot renders
- If denied or unavailable: map centers on `lat: 34.0522, lng: -118.2437` (Los Angeles default); no dot renders; distance filter disabled with label "Enable location to use distance filter"

### Mobile Bottom Sheet (Capacitor iOS only)

- Full-viewport map renders under the bottom sheet
- Sheet default snap point: 40% viewport height (trainer list visible)
- Sheet drag handles: 40% (default), 80% (full list), 10% (map-only, sheet mostly hidden)
- Sheet contains the same trainer list items as list view, filtered by map viewport
- Panning map while sheet is at 40%+ snaps sheet to 10% to give map focus

### Trainer Location Manager

- Located in trainer settings page under "Workout Locations" section
- Address field: Google Places Autocomplete, shows dropdown suggestions below input
- After selecting address: map preview renders centered on geocoded coordinates; drag-to-adjust pin active
- Drag complete: latitude/longitude updated to dropped position
- Location type: three-button toggle (Gym / Park / In-Home); required before saving
- Nickname field: optional text input, placeholder "e.g. Venice Beach Spot"; max 40 characters
- Maximum 5 locations: "Add Location" button disabled and shows tooltip "Maximum 5 locations reached" when at capacity
- Delete location: trash icon button (lucide `Trash2`, 16px, `aria-label="Remove location"`); triggers inline confirmation "Remove this location? This cannot be undone. [Cancel] [Remove]" — uses `red-500` for Remove button

### Go Live Location Picker

- Triggers as modal/sheet step when trainer taps the Go Live button in AvailabilityHeader
- Shows list of trainer's saved workout locations as selectable cards
- Each card: location icon + color dot, nickname or address, location type label
- If trainer has no saved locations: prompt "Add a workout location first" with link to settings
- Selecting a location and tapping "Go Live" completes the existing warm-up flow

## Map Styling

Primary visual anchor: the map surface with trainer pins. All overlay controls (chips, search button, bottom sheet) are secondary to the map canvas.

Map style: light, minimal — use Google Maps default light style. No custom map JSON styles this phase (left to Claude's discretion per CONTEXT.md). Avoid dark map theme as it conflicts with `--color-paper` dominant surface.

Pin dimensions:
- Individual pin: 28px wide × 36px tall SVG with location-type icon centered in upper body; location-type color fill; Elite: 2px `#C5A059` ring + star overlay top-right corner
- Live pulse overlay: 8px green-500 dot, `animate-pulse`, positioned bottom-right corner of pin

Info card dimensions:
- Width: 240px on desktop, 280px on mobile
- Padding: 16px all sides
- Border: 1px `ink/10`
- Shadow: `shadow-lg`
- Avatar: 40×40px circle, `object-cover`
- Position: anchored above the pin with 8px gap; repositions to below pin if pin is within top 30% of map viewport
